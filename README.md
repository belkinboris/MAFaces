# Реестр — MVP

Платформа о сделках и компаниях российского рынка. Демонстрационная версия: визуал, структура, вымышленные данные.

## Что внутри

- **Лента сделок** — фильтры по индустриям, статусы, суммы
- **Карточка сделки** — переключатель линзы: Обзор / Экономист / Юрист
- **Компании** — профили с показателями, лицами и историей сделок
- **Консультанты** — выдача по числу известных сделок, с дисклеймером (не рейтинг)
- **Ассистент** — демо-чат, отвечает по локальной базе со ссылками на карточки

Все данные вымышлены и зашиты в `static/index.html` (константы `DEALS`, `COMPANIES`, `FIRMS`) — редактируются прямо там.

## Локальный запуск

```bash
pip install -r requirements.txt
uvicorn main:app --reload
# http://127.0.0.1:8000
```

## Деплой на Railway

1. Запушить репозиторий на GitHub
2. Railway → New Project → Deploy from GitHub repo
3. Railway сам подхватит `Procfile` и `requirements.txt`, ничего настраивать не нужно
4. Healthcheck: `/health`

## Переименование

Название «Реестр» — рабочее. Меняется в `static/index.html`: `<title>`, класс `.wordmark`, подпись ассистента.

## Пайплайн наполнения базы (pipeline/)

Сбор сделок с сайтов фирм — три шага:

```bash
cd pipeline
export ANTHROPIC_API_KEY=sk-ant-...

# 0 (опционально, бесплатно): посмотреть, какие ссылки найдутся
python3 collect_deals.py --dry-run

# 1: собрать сделки (все фирмы или выборочно)
python3 collect_deals.py --firms orion,alrud,epam

# 2: ручной апрув — открой collected_deals.json, удали мусор, проверь low-confidence

# 3: конвертировать в формат сайта и вставить в MINI_DEALS (static/index.html)
python3 to_minideals.py collected_deals.json > mini_deals_snippet.js
```

Принципы: пауза 1,5 с между запросами к сайту; извлечение — claude-sonnet-4-6 (флаг `--model` позволяет удешевить до haiku); модель обязана возвращать только факты из текста статьи с URL источника; записи с confidence=low в сайт не попадают без ручной проверки. Ничего не публикуется автоматически.

Ориентир по стоимости: ~30 фирм × ~30 статей ≈ 900 вызовов ≈ $3–7 на Sonnet.
