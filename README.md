# Реестр — MVP

Платформа о сделках и компаниях российского рынка. Демонстрационная версия: визуал, структура, вымышленные данные.

## Что внутри

- **Лента сделок** — фильтры по индустриям, статусы, суммы
- **Карточка сделки** — переключатель линзы: Обзор / Экономист / Юрист
- **Компании** — профили с показателями, лицами и историей сделок
- **Консультанты** — выдача по числу известных сделок, с дисклеймером (не рейтинг)
- **Ассистент** — демо-чат, отвечает по локальной базе со ссылками на карточки

Все данные вымышлены и зашиты в `static/index.html` (константы `DEALS`, `COMPANIES`, `FIRMS`) — редактируются прямо там.

## Локальный запуск

```bash
pip install -r requirements.txt
uvicorn main:app --reload
# http://127.0.0.1:8000
```

## Деплой на Railway

1. Запушить репозиторий на GitHub
2. Railway → New Project → Deploy from GitHub repo
3. Railway сам подхватит `Procfile` и `requirements.txt`, ничего настраивать не нужно
4. Healthcheck: `/health`

## Переименование

Название «Реестр» — рабочее. Меняется в `static/index.html`: `<title>`, класс `.wordmark`, подпись ассистента.

## Пайплайн наполнения базы (pipeline/)

Три шага: сбор → дедуп+обогащение → конвертация в формат сайта.

### Шаг 1 — сбор сырья (сайты фирм + Telegram)

```bash
cd pipeline
export ANTHROPIC_API_KEY=sk-ant-...

python3 collect_deals.py --dry-run                         # бесплатная разведка
python3 collect_deals.py --firms orion,alrud,epam           # сайты фирм
python3 collect_deals.py --telegram dealsma,LawFirms --telegram-pages 10  # каналы
```

### Шаг 2 — дедуп и обогащение (enrich_deals.py)

Каналы вроде «M&A Новости» дают сотни сырых упоминаний, где одна сделка часто повторяется
несколько раз и описана скупо («сделка была», без сумм и консультантов). Этот шаг:
схлопывает дубли по ключу «стороны + дата ±30 дней», расставляет приоритет на самые «тонкие»
записи и дотягивает по ним детали через веб-поиск (сайты юрфирм, финансовых консультантов, СМИ).

```bash
# Сначала бесплатно посмотреть, сколько реально уникальных сделок после дедупа
python3 enrich_deals.py --in collected_telegram.json,collected_deals.json --dedupe-only

# Тестовый прогон на 30 самых приоритетных записях
python3 enrich_deals.py --in collected_telegram.json,collected_deals.json --limit 30

# Продолжить с чекпоинта (сохраняет после КАЖДОЙ записи — таймауты не страшны)
python3 enrich_deals.py --in collected_telegram.json,collected_deals.json --resume --limit 100
```

Результат — `enriched_deals.json`: для каждой сделки `draft` (исходная запись) и `enriched`
(что нашлось доп. — сумма, консультанты, детали, все со списком источников). Это тоже
материал для ручного апрува, не для прямой публикации.

### Шаг 3 — конвертация в формат сайта

```bash
python3 to_minideals.py collected_deals.json > mini_deals_snippet.js
```

Принципы на всех шагах: пауза между запросами; модель не имеет права выдумывать факты сверх
найденных источников; low-confidence/none-enrichment записи не идут в сайт без ручной проверки;
ничего не публикуется автоматически.

Ориентир по стоимости: сбор ~30 фирм × ~30 статей ≈ $3–7 на Sonnet; обогащение с веб-поиском
дороже за счёт тарифа на сам поиск (~$10/1000 запросов) — поэтому дедуп и приоритизация на
шаге 2 не формальность, а способ не платить за обогащение дублей и не тратить бюджет вслепую.
